核心能力 / Skills 驱动的能力复用

把分析方法、领域流程和工具调用沉淀为可复用技能

Skills 是 DB-GPT 的可扩展能力载体。它让团队可以把固定分析流程、脚本、说明和资源打包，然后通过前端上传、GitHub 导入或 Agent 调用复用。

API：/api/v1/skills/*前端：construct/skills执行：agentic_data_api.py

能力定位

方法沉淀

把 CSV 分析、财报分析、销售分析等固定流程沉淀为目录和脚本。

团队共享

通过上传或 GitHub 导入，让不同用户复用同一套技能。

Agent 调用

Agent 可以加载 skill 文件、执行脚本，并将输出写入分析过程。

实现链路

1. 技能发现/v1/skills/list 扫描内置和用户技能目录。
2. 查看详情/v1/skills/detail 返回文件树和文件内容。
3. 上传技能/v1/skills/upload 校验并安装 zip 包。
4. GitHub 导入/v1/skills/import_github 下载仓库并抽取技能目录。
5. Agent 执行通过 execute_skill_script 或工具函数执行技能脚本。

流程图：技能如何被安装、查看和调用

技能来源
内置目录、上传 zip、GitHub 仓库

系统同时支持内置技能和用户技能，GitHub 导入会下载并抽取目标目录。

技能登记
安装到用户技能目录

_install_skill_from_dir 处理目录复制、覆盖和基础校验。

前端查看
技能列表和文件树

skills/detail 返回树结构，前端可以浏览 README、脚本和资源。

Agent 加载
读取技能文件和脚本

load_skill、execute_skill_script 让 Agent 在分析过程中使用技能。

结果输出
技能脚本输出进入分析流

脚本结果被作为工具输出或上下文，参与最终回答。

如何实现：源码分层拆解

1. API 管理层

agentic_data_api.py 提供 list/detail/upload/import/download 等技能管理接口。

2. 技能安装层

_install_skill_from_dir、_extract_skill_from_zip、_parse_github_url 处理安装来源。

3. 技能执行层

execute_skill_script、execute_skill_script_file 负责执行技能内脚本。

4. 前端管理层

web/pages/construct/skills/index.tsx 负责列表、详情、上传、GitHub 导入弹窗。
