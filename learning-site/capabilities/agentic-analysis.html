核心能力 / Agentic 数据分析

让数据分析从“问答”升级为“计划、执行、校验”的 Agent 流程

Agentic 数据分析是 DB-GPT 的中枢能力。它把用户的自然语言目标转成可执行步骤，结合数据源、知识库、工具、代码解释器和模型推理，逐步产出表格、图表、报告或结论。

入口：/api/v1/chat/completions 场景：ChatAgent / ChatFlow / ChatDashboard 核心：ConversableAgent / ReActAgent

能力定位

这项能力解决的是开放式分析任务：用户给出目标，系统需要自己决定查什么数据、调用什么工具、怎样解释结果。

输入不是指令，而是目标

例如“分析销售下降原因”，不是单条 SQL，也不是固定报表。Agent 需要理解目标并拆解成可执行动作。

输出不是文本，而是结果包

答案可能包含 SQL 查询、Python 代码、图表、文件、工具日志、阶段性思考和最终报告。

执行过程可观察

agentic_data_api.py 会把步骤、工具输出和最终响应包装为流式事件，前端可逐步展示进度。

实现链路

Agentic 分析不是单点功能，而是 API、场景、Agent、工具、回调和前端流式渲染协作的结果。

1. 用户请求前端通过 useChat 或 react-agent 接口发送任务目标。
2. 场景判断api_v1.py 根据 ChatScene 选择普通聊天、Agent、Flow 或知识库。
3. Agent 构建ConversableAgent 初始化上下文、资源、LLM 配置和动作。
4. 执行动作thinking → act → verify 循环调用 SQL、代码、知识检索或技能。
5. 流式返回步骤事件、工具输出和最终答案通过 SSE 返回前端。

前端发起
Chat 页面或 Agentic 页面提交任务目标

前端把用户输入、会话 ID、模型参数、资源选择和 app 信息打包，通过 SSE 接口发给后端。

API 分流
chat_completions / chat_react_agent 决定执行路径

普通 chat 走 api_v1.py 的 chat_completions；更强的 agentic data 流程走 agentic_data_api.py 的 chat_react_agent。

构造 Agent
加载上下文、资源、LLM 配置和工具

ConversableAgent.build() 会准备系统提示词、资源变量、记忆和动作列表，为后续思考和行动做准备。

执行循环
thinking → act → verify

Agent 先生成思考或计划，再通过工具执行 SQL、代码、知识检索、技能脚本等动作，最后校验是否得到可用结果。

流式展示
步骤、工具输出、最终答案逐步回到 UI

stream_callback 和 SSE event 将中间步骤发给前端，用户能看到过程而不只是最终结果。

如何实现：源码分层拆解

这项能力由三层组成：API 编排层、Agent 抽象层、工具执行层。

1. API 编排层

api_v1.py 负责传统聊天场景，agentic_data_api.py 负责更复杂的 ReAct/Manus 风格执行流，包括步骤构造、工具输出分块和下载接口。

2. Agent 抽象层

agent.py 定义 Agent 必须具备的生命周期；base_agent.py 实现上下文管理、模型调用、动作调用和结果校验。

3. 具体 Agent 层

react_agent.py 和 data_analysis_agent.py 在抽象 Agent 基础上实现工具推理和数据分析任务。

4. 工具执行层

agentic_data_api.py 内部注册了知识检索、SQL 查询、代码解释器、HTML 解释器、shell 解释器和 skill 执行等工具函数。

二次开发切入点

如果要扩展 Agentic 分析，可以从以下位置入手。

新增工具

在 agentic_data_api.py 的工具函数组里增加可调用函数，并处理输出分块、错误和展示格式。

新增 Agent

继承 ConversableAgent 或参考 DataAnalysisAgent，重写资源准备、act 参数和记忆片段。

调整步骤展示

关注 build_step、step_chunk、stream_callback，这些影响前端如何展示执行过程。

接入新资源

把数据库、知识库、文件或外部工具包装成 Agent 可用资源，供 load_resource 或 tool action 调用。

学习这项能力时，重点不是背函数名，而是掌握一个模型：Agent 接收目标，构造上下文，选择动作，执行工具，校验结果，再把过程流式返回。
