核心能力 / RAG 知识库

把企业文档转成模型可检索、可引用的上下文

RAG 知识库能力负责知识空间管理、文档上传、切分、embedding、向量存储、召回测试和知识增强回答。

API：/knowledge/*核心：KnowledgeService场景：ChatKnowledge

能力定位

RAG 的目标是解决模型无法知道企业私有知识的问题。DB-GPT 将文档处理成可检索片段，并在回答时注入上下文。

知识空间

按业务、项目或团队创建 space，隔离不同知识集合。

文档处理

上传文档后执行切分、保存、embedding 和同步。

召回与回答

根据用户问题召回相关 chunks，再结合模型生成答案。

实现链路

1. 创建 Space/knowledge/space/add 创建知识空间。
2. 上传文档/document/upload 保存文件并登记文档。
3. 文档同步/document/sync 触发切分和 embedding。
4. 召回测试/recall_test 验证 retriever 策略。
5. 知识聊天ChatKnowledge 使用检索结果增强回答。

流程图：文档如何变成可问答知识库

文件进入知识空间

chunk + embedding

召回测试
验证检索效果

recall_test 可以检查 query 是否能召回正确片段。

知识聊天
检索结果注入 prompt

ChatKnowledge 根据用户问题召回 chunks，把片段作为上下文交给模型回答。

如何实现：源码分层拆解

1. Knowledge API 层

knowledge/api.py 暴露 space、document、chunk、recall、summary 等接口。

2. KnowledgeService 层

KnowledgeService 承载业务逻辑，包括创建空间、文档同步、chunk 查询和 summary。

3. RAG Core 层

dbgpt/rag 负责文档加载、切分、assembler、retriever 和 embedding 组合。

4. Storage 扩展层

dbgpt_ext/storage 负责接入 Chroma、Milvus、Qdrant 等向量存储。
