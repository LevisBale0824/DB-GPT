核心能力 / 沙箱代码执行

让模型生成的代码在受控环境中运行

DB-GPT 的 sandbox 包定义了统一的 SandboxRuntime 和 SandboxSession 抽象，支持 local、docker、podman、nerdctl 等执行后端，用于隔离运行 Python、bash 或分析脚本。

包：dbgpt-sandbox抽象：SandboxRuntime / SandboxSession调用：agentic_data_api shell_interpreter

能力定位

隔离执行

模型生成代码需要运行，但不应该直接影响主服务进程和宿主环境。

统一接口

无论 local、docker 还是 podman，都通过 SandboxRuntime 和 SandboxSession 抽象调用。

资源限制

执行层可以设置工作目录、超时、内存和环境变量，降低失控风险。

实现链路

1. Agent 生成代码分析任务需要运行 Python 或 shell。
2. 调用解释器code_interpreter 或 shell_interpreter 接收代码。
3. 创建 SessionLocalRuntime.create_session 创建临时执行目录。
4. 执行并限制execute 写入脚本文件并通过子进程执行。
5. 返回结果ExecutionResult 包含 stdout、stderr、状态和耗时。

stdout、stderr、exit code、耗时被封装后交还给 Agent 或前端。

如何实现：源码分层拆解

1. 抽象层

base.py 定义 ExecutionResult、SessionConfig、SandboxSession、SandboxRuntime。

2. 本地执行层

local_runtime.py 创建临时目录、写入代码文件、生成执行命令并收集输出。

3. 容器执行层

docker_runtime.py、podman_runtime.py、nerdctl_runtime.py 提供更强隔离方案。

4. Agent 调用层

agentic_data_api.py 的 shell_interpreter 直接展示了如何调用 dbgpt-sandbox。
