核心能力 / 可视化数据智能控制台

把后端 AI 能力变成可配置、可调试、可复用的产品界面

DB-GPT Web 不只是聊天页，它还提供数据源、知识库、模型、Prompt、Flow、Skills、应用构建和分享等管理页面，让数据智能能力可操作、可维护。

前端：Next.js + React聊天：useChat + fetchEventSource管理：web/pages/construct

能力定位

统一入口

用户通过 Web UI 使用聊天、数据源、知识库、Skills 和模型配置。

流式交互

useChat 基于 fetchEventSource 接收后端流式响应。

管理控制台

construct 页面族承载应用构建、数据库、知识库、模型、技能、工作流等管理功能。

实现链路

1. 页面入口web/pages/chat/index.tsx 读取路由参数和上下文。
2. Hook 封装useChat 封装请求、SSE、abort 和状态回调。
3. API 请求请求拼接 API_BASE_URL 和后端接口。
4. 消息渲染前端按流式 chunk 更新聊天内容、图表和工具输出。
5. 控制台管理construct 页面通过 axios/fetch 调用数据源、知识库、技能等 API。

流程图：前端如何把能力组织成产品

页面路由
Next.js pages 定义产品入口

web/pages/chat 是聊天主界面，web/pages/construct 是管理控制台。

状态与 Hook
上下文和 hooks 组织交互状态

ChatContext 管理当前模型和会话，useChat 管理流式请求。

API 调用
axios / fetchEventSource 调后端

聊天走 SSE，管理页多用 axios/fetch 调用普通 REST API。

渲染结果
消息、表格、图表、文件和工具日志

前端根据后端返回的不同内容类型渲染不同组件。

管理闭环
配置数据源、知识库、模型、技能

construct 页面把后端能力暴露成可配置、可维护的产品功能。

如何实现：源码分层拆解

1. 页面层

web/pages 负责路由入口，Chat、construct、share、mobile 都从这里进入。

2. 组件层

web/components 和 web/new-components 承载聊天消息、布局、面板、编辑器等 UI。

3. 数据层

web/hooks 和 web/client 封装请求、流式响应和业务状态。

4. 后端映射层

每个页面最终都会落到后端的 /api 或 /knowledge 路由，应通过接口路径反查 Python router。
