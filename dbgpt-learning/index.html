DB-GPT 源码精读 v0.8.1

一、工程定位

核心能力四件套：

  • Agent —— 自主规划、调用工具（ReAct / AutoGen 风格）
  • AWEL —— Agentic Workflow Expression Language，DAG 编排（DB-GPT 灵魂）
  • RAG —— 知识检索（Knowledge → Embedding → VectorStore → Retriever → Reranker）
  • 多源数据访问 + 沙箱执行 —— 数据库/CSV/知识库 + 隔离 Python 执行

二、Monorepo 包结构

使用 uv workspace 管理 7 个包：

# pyproject.toml (顶层)
[tool.uv.workspace]
members = [
  "packages/dbgpt-app",
  "packages/dbgpt-client",
  "packages/dbgpt-core",
  "packages/dbgpt-ext",
  "packages/dbgpt-serve",
  "packages/dbgpt-sandbox",
  "packages/dbgpt-accelerator/dbgpt-acc*",
]}
包角色徽章
dbgpt-core核心抽象与接口（awel/agent/rag/model/datasource 全在这里定义）core
dbgpt-ext扩展实现（各 LLM、数据源、向量库、可视化）ext
dbgpt-serveREST 服务层（agent/conversation/flow/model/rag/...）serve
dbgpt-app应用编排层（FastAPI 主入口 + Scene + openapi）app
dbgpt-clientPython SDK（外部系统集成）app
dbgpt-sandbox代码沙箱执行ext
dbgpt-accelerator推理加速（dbgpt-acc-auto / dbgpt-acc-flash-attn）ext

三、包依赖图与依赖倒置

flowchart TD APP["dbgpt-app
应用编排层
(FastAPI/Scene)"] SERVE["dbgpt-serve
REST 服务"] EXT["dbgpt-ext
扩展实现"] CORE["dbgpt-core
核心抽象/接口"] CLIENT["dbgpt-client
外部 SDK"] ACC["dbgpt-accelerator
推理加速"] SANDBOX["dbgpt-sandbox
代码隔离执行"] APP --> SERVE APP --> EXT APP --> CORE SERVE --> EXT SERVE --> CORE EXT --> CORE CLIENT -.-> SERVE ACC -.-> CORE SANDBOX -.-> CORE
依赖倒置原则：app/serve 依赖 core 的抽象，ext 提供实现， 运行期由 component_configs.py 把实现注入 SystemApp 容器。 这就避免了高层模块直接依赖低层实现，破解了循环依赖。

六、核心技术问题清单

读每个模块时，带着下面的问题去源码里找答案：

AWEL（必答）

  • DAG 怎么编译？拓扑排序在哪？
  • 流式输出（streaming）怎么在 DAG 节点之间传递？
  • Operator 之间的数据契约是什么？
  • UI 编排的 JSON 怎么反序列化成 DAG 对象？

Agent

  • ReAct 推理循环在哪？怎么终止？
  • 工具调用（function call）是怎么发起和解析的？
  • 多 Agent 协作（multi-agent）怎么编排？
  • Memory 怎么存？短期 vs 长期？

RAG

  • Knowledge 怎么分块（chunk）？有哪些策略？
  • 不同 Embedding 模型怎么热插拔？
  • Retriever 怎么和 Agent 联动？

Model / DataSource

  • proxy 模式 vs local 模式有什么本质区别？
  • 多 worker 怎么负载均衡？
  • 不同数据库方言怎么抽象？schema 怎么缓存给 LLM？
说明：本手册采用精读关键文件 + 概览全貌的方式。 DB-GPT 工程代码量巨大（数百文件、数万行），不可能"逐行"全部解读。 每章会精选 5~15 个核心文件，对关键代码段做逐行/逐段精读，其余文件给出概览和定位说明。
